A reflection on Tim Berners Lee's: The World Wide Web.

"The world wide web first began life at CERN, a European laboratory for Particle Physics in Geneva, Switzerland".
(Berners-Lee, 1994)

Berners- Lee, the inventor of the web, came up with a solution to aid the dissemination of research and information amongst his peers and the critical mass. As such he came up with a distributed network that allows researchers from different parts of the world to organize and efficiently gather information from multiple sources.

The web was designed, with the idea of creating a distributed hypertext system that affords users with the ability to work independently. This design is reflected and illustrated by version control systems, namely git and git hub, which provides a platform or server within which a team member can independently work on a team project from remote locations around the world.

Typically a client (browser) is used to request information off a server (a computer that runs the back-end of the internet) which, is used to store the metadata of any data that is represented on the internet. The server then responds to the clients request by retrieving an html document, which it interprets and describes to the client in order to render multimedia information.

Moreover the web uses an address system, network protocol html to process and respond to a clients request. The first way tThis address system is called a URL - Universal Resource Identifiers, and is used to retrieve and embed the ip address of a desired repository or site as a domain name, so that a user can find and address any object as the set of protocol parameters necessary to access the document.

What is Semantic Markup? Why is it necessary?

The word "Semantics" is a branch of semiotics which interprets the representational value of signs and symbols through the study of meanings. Therefore in programming, Semantics refers to the syntactical meaning of a piece of code within a given document.

To give a document meaning is to render it usable and responsive to the contextual needs of any user receiving information from a browser. If the browsers contextual needs are met, the browser will be able to render all the documents needed to structure, and embed multimedia faster, to optimize the performance of the web so that every user is able to "experience" the web.Another reason would be for the optimization of web searches, and monitoring of web services and sites.

What is a browser and how does it work?

A browser is a client - side programme that hosts users requests and responds with various multimedia elements (texts, images, audio, and videos). This occurs whenever a user requests a document by entering a URL, clicking a link or submitting a form. As such the Semantic mark up of the page becomes paramount to the prospects of minimizing the delay between client - server interaction.

The browser starts by first identifying the markup of a given page and attempts to find where the embedded multimedia elements are located. For secure transactions to happen within this process a TLS negotiation needs to take place wherein this negotiation determines how to encrypt communication, verifies the server and establishes a secure connection between the client and the server so that the user does not become exposed to any malicious intent. After a connection is established parsing takes place, this is a step the browser takes to turn the data it receives over the network and into the Dom, which is then used to render a web pages layout, paint, style and compositing described within an html and the DOM tree - a DOM tree describes the content of a document, and reflects the different sematic, stylistic relationships and hierarchies a document has to offer.

How does the Medium is the Message impact our understanding of the web

In the book: The medium is the message Marshal McLuhan writes that

"The medium is the message because it is the medium that shapes and controls the scale and form of human association and action"
(McLuhan, 1964)

To understand what this means we have to first understand what medium and message mean. Medium, refers to the tools or instruments used to provide agency to a user. Message refers to the contextual underpinnings that tools use to cultivate meaning through the use of semiotics such as text, images, and audio all of which are hypermedia which provide an extension outside the traditional analogue media constraints.

With this knowledge we are able to understand the world wide web as a mechanic that functions as an interconnected continuum of inter-textual meanings created by the ability for the medium to have an adverse effect on the scale and form of human action and association, through the structure and configuration of hypermedia and hyperlinking it becomes clear that the message is the non-linear reception and reciprocation of threads of information between a network of people sharing and retrieving information on a distributed network.
